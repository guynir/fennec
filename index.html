What Templating is all about

Templating is the art of taking an unprocessed text, adding it a context and generating contents.

The template: Welcome back Mr. {{lastName}}.
The context: lastName = "Bond"
The outcome: Welcome back Mr. Bond.

Why do I need templating ?

Because sometimes your application must be dynamically generated in run-time:

<body>
    Welcome back, Mr. {{lastName}}.<br>
    We have been expecting your.<p>

    The last time you where here was {{lastLoginTime}}.
</body>

In the example above, the two tokens (or variables) lastName and lastLoginTime changes with each user and login time. These values must be dynamically generated.

Why Fennec ?

The Fennec library gives the developer an easy way to tie between a client-side templating library (currently only Handlebars is supported) and a DOM element. It contains the boilerplate code you'd probably write yourself in every project that uses client-side templating.

The example

(I removed some elements for better clarity).

<head>
    <script type="text/javascript">
        // Reference to agent.
        var agent;
        $( document ).ready(function() {
            // Create a new template agent.
            agent = new TemplateBinder(".container");
            agent.refresh();
        });

        function applyNewName(theName) {
            agent.refresh({name: theName});
        }
    </script>
</head>

<body>
    <div class="container">
        <span>
            {{#if name}}
                Hello {{name}} ! Very pleased to meet you !!!
            {{else}}
                Hello Mr., I don't know your name.
            {{/if}}
        </span>
        <form id="form">
            <label>
                {{#if name}}
                You can change your name:
                {{else}}
                Please tell me your name:
                {{/if}}
            </label>
            <input type="text" id="name">
            <input type="button" onclick="applyNewName(this.form.name.value)" value="Change">
        </form>
    </div>
</body>

The body itself contains a simple template in a syntax supported by Mustache and Handlebars.

The idea is simple: Every time we call applyNewName() with a new string representing a name, the body is rendered and new values are applied. You can download the code from this example at GitHub.
